# 德州扑克 UI 全面升级需求（严格按截图 6 人环形席位实现）

## 一、整体项目基础约束（继承之前全部开发要求，不可修改）
1. 开发框架：Nuxt 4，使用 Vercel KV（Redis）云端持久化存储房间、对局全部数据，解决玩家掉线丢失游戏进度问题；后端 Nitro 编写所有游戏核心逻辑，前端仅负责页面渲染与操作提交，防止前端篡改游戏数据。
2. 适配要求：仅开发手机竖屏浏览器版本，完全放弃 PC 端适配，电脑端打开页面直接弹窗提示「请使用手机浏览器访问」。
3. 部署要求：项目遵循 Nuxt 4 标准目录结构，配置 Vercel KV 环境变量，关联 GitHub 仓库后可一键全自动部署到 Vercel 公网。
4. 原有业务规则完整保留，UI 只做视觉层替换：
   - 房间依靠线下私密密码创建/加入，首位创建者为房主，单房间总人数上限 6 人（真人+机器人共同占用席位）；
   - 房主等待阶段可在空位添加 AI 机器人，点击开局后房间永久锁人，新用户无法凭密码进入；
   - 全员初始筹码固定 3000，盲注固定小盲 10、大盲 20；
   - 玩家轮到行动时开启 60 秒倒计时，超时系统自动判定弃牌；
   - 玩家掉线后凭借本地会话令牌可断线重连，从 Vercel KV 读取完整对局快照恢复游戏。

## 二、游戏界面整体场景视觉（完全参照截图复刻）
1. 背景场景：高端城市观景包厢，落地窗可以看到夜晚城市灯光天际线，窗户两侧悬挂深蓝色丝绒窗帘；室内搭配轻奢小圆桌、红酒瓶酒杯、休闲沙发座椅，地面带有黑桃花色地毯纹样，整体是二次元写实卡通手游画风。
2. 扑克桌面：椭圆形德州专用牌桌，桌面主体为深蓝色，桌边缘是黑色软包+深色包边设计，桌面印有浅灰色半透明 HOLD'EM 字样；
   - 桌面顶部正中间固定摆放卡通女性荷官形象，荷官手持红色牌组负责发牌，位置永久固定不动；
   - 荷官正下方区域展示本局底池总筹码金额（例如 1.5M 格式）；
   - 桌面左上角摆放庄位标识「D」圆形筹码片，旁边附带红色牌背卡片；
3. 公共牌展示区：桌面水平居中位置横向摆放对局公共纸牌（翻牌 3 张、转牌第 4 张、河牌第 5 张依次展示），纸牌带有实体纸张光影、阴影质感，还原线下扑克牌视觉效果。

## 三、核心 6 人环形席位布局（硬性规则）
一共 **6 个玩家席位**，玩家本人**永久固定在【下方左侧席位】，不可变动**。6 个席位沿椭圆牌桌顺时针均匀环绕排布：

1. **席位 1（玩家本人固定位，页面下方左侧，永久锁定）**
   - 展示圆形卡通个人头像，头像右上角带有紫色礼物礼盒小图标；
   - 头像下方展示自身当前剩余筹码金币数值；
   - 头像正前方固定展示自己的两张底牌，底牌下方显示当前手牌牌型名称（如高牌、一对），手牌下方搭配彩色胜率进度条；
   - 轮到本人行动时，界面会同步展示 60 秒行动倒计时可视化进度。
2. **席位 2**：页面右下角位置（环形右下方）
3. **席位 3**：页面右上角位置（环形右上方）
4. **席位 4**：桌面顶部右侧，荷官右手边位置
5. **席位 5**：桌面顶部左侧，荷官左手边位置
6. **席位 6**：页面左下角位置（玩家本人左侧空位）

### 所有席位统一 UI 规范
1. 所有真人玩家、AI 机器人全部使用圆形头像框展示，头像右上角统一标配紫色礼物礼盒图标；
2. 头像下方展示该玩家剩余筹码数量，高等级玩家可额外佩戴皇冠、徽章特效装饰；
3. 除玩家本人以外，其余 5 个席位玩家的手牌全程默认背面遮挡，界面标注「暂时无法查看」，只有对局摊牌结算阶段才统一亮牌展示手牌；
4. 玩家执行全下、下注操作后，座位侧边会同步展示红色牌背卡片，标记操作状态；
5. AI 机器人占用空位席位时，严格按照 6 个环形空位顺序填充席位，布局和真人玩家完全一致。

## 四、页面左侧固定竖向功能栏（手机屏幕最左边缘从上至下排列）
1. 左上角第一个：紫色方形返回箭头按钮，页面左上角同步显示系统时间（如 07:51）；
2. 第二个：紫色方形聊天对话框按钮，点开可打开房间对局聊天窗口；
3. 第三个：圆形红色靶心功能图标；
4. 第四个：方形增益 BUFF 图标，图标内标注「5%」「1X」标识。

## 五、页面底部对局操作按钮区
参照截图样式，横向排列 4 个深蓝色圆角操作按钮，按钮内为白色中文文字，从左到右依次为：
**【弃牌】、【过牌/弃牌】、【过牌】、【跟任意注】**

交互规则：系统根据当前对局下注轮次状态，自动判断按钮是否可点击，不可操作的按钮置灰禁用；玩家点击按钮后操作提交后端 Vercel KV，前端同步刷新对局状态。

## 六、界面交互附加细节要求
1. 所有头像、按钮、纸牌、特效全部添加柔和投影、微光发光效果，提升手游商业化立体感；
2. 金币、筹码统一使用金黄色视觉样式，数值支持 M（百万）缩写简化展示；
3. 对局操作日志在界面侧边竖向滚动展示，记录全部下注、弃牌、机器人操作、超时自动弃牌、结算记录；
4. 界面保留断线重连快捷按钮，玩家掉线重开页面后，点击按钮读取本地会话令牌，一键从云端 KV 恢复对局。

## 七、开发交付要求
1. **UI 视觉代码和原有业务逻辑代码解耦，只替换界面渲染层**，房间、数据库、对局规则代码完全保留不变；
2. 项目根目录生成完整 README.md 部署文档，包含 Vercel KV 配置、GitHub 部署、本地启动调试命令；
3. 核心代码添加详细注释，后续可自主修改头像样式、筹码数值、界面配色。

## 执行要求
- 严格按照上面规范实现，不要修改后端逻辑与 KV 数据结构。
- 席位映射：本地玩家在 KV 中的 seat index 可能任意，但**渲染时永久映射到席位 1（下方左侧）**，其他玩家按顺时针依次填入席位 2→6。
- 完成后自测 `pnpm build`（或 npm）通过，然后 `git add -A && git commit && git push`，Vercel 自动部署。
- 若发现原有业务逻辑漏洞可以修，但需在提交信息里单独列出。
